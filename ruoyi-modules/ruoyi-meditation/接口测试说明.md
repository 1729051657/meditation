# 接口测试说明

## 新增接口测试

### 检查收藏状态接口

#### 接口信息
- **接口路径**: `GET /meditation/favorite/check`
- **请求方式**: GET
- **接口描述**: 检查用户是否已收藏指定内容

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| trackId | Long | 是 | 目标内容ID |
| type | String | 是 | 目标类型，支持：track、series、article |

#### 测试用例

##### 1. 检查音频收藏状态
```bash
GET /meditation/favorite/check?trackId=301&type=track
```

**预期响应**:
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": true
}
```

##### 2. 检查系列收藏状态
```bash
GET /meditation/favorite/check?trackId=101&type=series
```

**预期响应**:
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": false
}
```

##### 3. 检查文章收藏状态
```bash
GET /meditation/favorite/check?trackId=201&type=article
```

**预期响应**:
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": false
}
```

#### 错误情况测试

##### 1. 缺少必要参数
```bash
GET /meditation/favorite/check?trackId=301
```

**预期响应**:
```json
{
  "code": 400,
  "msg": "参数错误",
  "data": null
}
```

##### 2. 未登录用户
```bash
GET /meditation/favorite/check?trackId=301&type=track
```

**预期响应**:
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": false
}
```

#### 测试工具
可以使用以下工具进行测试：
1. **Postman** - API测试工具
2. **浏览器** - 直接访问URL
3. **curl命令** - 命令行测试

#### curl测试命令
```bash
# 测试音频收藏状态
curl -X GET "http://localhost:8080/meditation/favorite/check?trackId=301&type=track" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 测试系列收藏状态
curl -X GET "http://localhost:8080/meditation/favorite/check?trackId=101&type=series" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 注意事项
1. 需要用户登录状态（获取用户ID）
2. 参数类型必须正确（trackId为数字，type为字符串）
3. 返回值为布尔类型（true/false）
4. 接口支持多租户
