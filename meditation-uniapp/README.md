# EduGuard-Backend-Uniapp 校园安全巡检小程序

本目录为 **校园安全管理平台** 的「巡检端」小程序，基于 **uni-app** + **tuniao-ui** 开发，主要面向学校巡检员、保安等内部角色，用于校园安全巡检打卡和日常安全管理。

## 项目概述

校园安全管理系统是连接县级教育主管部门（简称"县局"）与辖区内各学校，专注于校园安全管理的任务协同、文件流转、资料归档与过程监管的数字化平台。

### 核心功能

1. **学校选择** - 支持多学校管理，巡检员可选择要管理的学校
2. **微信授权登录** - 通过微信授权获取手机号进行身份验证
3. **巡检打卡** - 支持位置打卡、人脸核验等功能
4. **安全任务** - 查看和处理安全相关任务
5. **文件管理** - 查看安全相关文件和通知
6. **数据统计** - 查看巡检数据和安全统计

## 开发方式一：HBuilderX 可视化

1. 安装 [HBuilderX](https://www.dcloud.io/hbuilderx.html) 最新版本，并确保安装 `uni-app` 插件。
2. 使用"文件 -> 导入 -> 导入 uni-app 项目"将本目录导入。
3. 在左侧选择运行平台（微信小程序 / H5 / App 等）即可真机预览或模拟器调试。
4. 若需发布微信小程序，请在 `manifest.json` 中填写 AppID 并勾选对应平台。

## 开发方式二：命令行 (CLI)

```bash
npm install -g @dcloudio/uni-app-cli   # 安装 uni-app CLI
npm install                            # 安装依赖
npm run dev:h5                         # H5 预览
npm run dev:mp-weixin                  # 微信小程序预览
```

> **注意**：当前模板的 `package.json` 仅包含基本信息，如使用 CLI 方式，请根据需要补充脚本和依赖。

## 接口与鉴权

- 所有接口调用统一封装于 `utils/request.js`。
- 默认后端地址在 `common/config.js` 中配置，可根据环境切换。
- 登录成功后，`token` 会被存储到 `store` 并在每次请求中携带。
- 支持租户隔离，不同学校的数据相互独立。

## 登录流程

1. **选择学校** - 用户首先选择要管理的学校
2. **微信授权** - 点击微信授权登录按钮
3. **手机号授权** - 微信弹出手机号授权
4. **后端验证** - 后端验证微信授权和手机号
5. **用户匹配** - 根据手机号查找系统中的用户
6. **登录成功** - 生成token并跳转到首页

## 常见问题

1. **微信登录失败**：请在微信开放平台配置合法域名，并确保后台已开启小程序端口。
2. **接口 401 未授权**：检查 `token` 是否过期或请求头是否携带。
3. **学校选择失败**：确保用户已在系统中注册，且手机号与系统记录一致。
4. **平台兼容性**：uni-app 默认支持 H5/App/各小程序，如需特定平台适配，请参考官方文档。
5. **巡检打卡失败**：检查位置权限和网络连接状态。
