# 存储键名使用说明

## 概述

为了避免硬编码字符串，提高代码的可维护性，我们创建了全局常量来管理所有的存储键名。

## 常量文件位置

- 主文件：`/common/constants.js`
- 导出文件：`/common/config.js`

## 使用方法

### 1. 导入常量

```javascript
// 方式1：从config.js导入（推荐）
import { STORAGE_KEYS } from '@/common/config.js'

// 方式2：直接从constants.js导入
import { STORAGE_KEYS } from '@/common/constants.js'
```

### 2. 使用常量

```javascript
// 旧方式（不推荐）
uni.setStorageSync('edu_token', token)
uni.getStorageSync('edu_token')

// 新方式（推荐）
uni.setStorageSync(STORAGE_KEYS.TOKEN, token)
uni.getStorageSync(STORAGE_KEYS.TOKEN)
```

## 可用的存储键名

### 用户认证相关
- `STORAGE_KEYS.TOKEN` - 用户认证令牌
- `STORAGE_KEYS.USER_INFO` - 用户信息
- `STORAGE_KEYS.REFRESH_TOKEN` - 刷新令牌

### 租户和部门相关
- `STORAGE_KEYS.TENANT_ID` - 租户ID
- `STORAGE_KEYS.DEPT_ID` - 部门ID

### 微信相关
- `STORAGE_KEYS.OPENID` - 微信openid
- `STORAGE_KEYS.SCOPE` - 用户scope

### 应用设置相关
- `STORAGE_KEYS.APP_SETTINGS` - 应用设置
- `STORAGE_KEYS.THEME` - 主题设置
- `STORAGE_KEYS.LANGUAGE` - 语言设置

### 缓存相关
- `STORAGE_KEYS.CACHE_DATA` - 缓存数据
- `STORAGE_KEYS.SEARCH_HISTORY` - 搜索历史

### 其他
- `STORAGE_KEYS.FIRST_LAUNCH` - 首次启动标记
- `STORAGE_KEYS.VERSION` - 版本信息

## Vuex 使用说明

### Actions（异步操作）

用户模块提供了以下 actions：

```javascript
// 导入 actions
import { mapActions } from 'vuex'

export default {
  methods: {
    ...mapActions('user', ['wxLogin', 'logout', 'getUserInfo', 'getToken'])
  }
}
```

- `wxLogin(params)` - 微信小程序登录
- `logout()` - 用户登出
- `getUserInfo()` - 获取用户信息
- `getToken()` - 获取用户token

### Getters（计算属性）

用户模块提供了以下 getters：

```javascript
// 导入 getters
import { mapGetters } from 'vuex'

export default {
  computed: {
    ...mapGetters('user', ['isLogin', 'token', 'userInfo', 'permissions', 'roles'])
  }
}
```

- `isLogin` - 登录状态
- `token` - 用户token
- `userInfo` - 用户信息
- `permissions` - 用户权限列表
- `roles` - 用户角色列表
- `tenantInfo` - 租户信息
- `openid` - 微信openid
- `scope` - 用户scope

### State（状态）

用户模块的状态：

```javascript
// 导入 state
import { mapState } from 'vuex'

export default {
  computed: {
    ...mapState('user', ['isLogin', 'token', 'userInfo', 'openid', 'scope'])
  }
}
```

## 注意事项

1. 所有存储键名现在都使用 `meditation_` 前缀，替代了原来的 `edu_` 前缀
2. 使用常量可以避免拼写错误，提高代码质量
3. 如果需要添加新的存储键名，请在 `constants.js` 文件中添加，并在文档中说明用途

## 登录参数配置

### 全局配置参数

登录相关的参数现在都从全局配置中获取：

```javascript
// 在 common/config.js 中配置
export const appConfig = {
  // 微信小程序AppID
  wechatAppId: 'wx1ef177b90f6fb543',
  
  // 客户端配置
  clientId: 'wx1ef177b90f6fb543',
  clientKey: 'user_xcx',
  
  // 默认租户ID
  defaultTenantId: '000000'
}
```

### 登录参数使用

```javascript
// 在 store/modules/user.js 中使用
const loginData = {
  xcxCode: code,
  appid: params.appid || appConfig.wechatAppId,        // 使用全局配置
  clientId: params.clientId || appConfig.clientId,     // 使用全局配置
  grantType: 'xcx',
  tenantId: params.tenantId || appConfig.defaultTenantId // 使用全局配置
}
```

### 参数优先级

1. **传入参数** - 如果调用时传入了参数，优先使用传入的值
2. **全局配置** - 如果没有传入参数，则使用全局配置中的默认值
3. **硬编码值** - 已完全移除，所有值都从配置中获取

## 示例代码

```javascript
// 设置用户token
uni.setStorageSync(STORAGE_KEYS.TOKEN, token)

// 获取用户token
const token = uni.getStorageSync(STORAGE_KEYS.TOKEN)

// 删除用户token
uni.removeStorageSync(STORAGE_KEYS.TOKEN)

// 检查是否已登录
const isLoggedIn = !!uni.getStorageSync(STORAGE_KEYS.TOKEN)
```
