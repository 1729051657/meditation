## 技术实现

### 1. 音频管理器
使用 `uni.getBackgroundAudioManager()` 替代普通音频上下文，确保后台播放支持。

### 2. 事件监听
监听音频播放的各种事件，包括播放、暂停、停止、上一曲、下一曲等。

### 3. 状态同步
保持小程序界面和后台播放状态的一致性，确保用户体验流畅。

### 4. 资源清理
在页面卸载时正确清理音频资源，避免内存泄漏和错误：

```javascript
// 清理音频资源 - BackgroundAudioManager 没有 destroy 方法
cleanupAudioResources() {
  // 清理定时器
  if (this.updateInterval) {
    clearInterval(this.updateInterval)
  }

  // 保存播放进度
  this.savePlayProgress()

  // 清理音频资源
  if (this.audioContext) {
    try {
      // 停止播放
      this.audioContext.stop()
      // 移除所有事件监听器
      this.audioContext.offPlay()
      this.audioContext.offPause()
      this.audioContext.offTimeUpdate()
      this.audioContext.offEnded()
      this.audioContext.offError()
      this.audioContext.offCanplay()
      this.audioContext.offWaiting()
      this.audioContext.offStop()
      this.audioContext.offPrev()
      this.audioContext.offNext()
      // 注意：不要清空 src，避免 invalid BackgroundAudioManager.src 错误
    } catch (error) {
      console.error('清理音频资源时出错:', error)
    }
  }
}
```

### 5. 生命周期管理
- `onUnload`: 页面卸载时智能选择清理策略
- `onHide`: 页面隐藏时保存播放进度，保持后台播放
- `onShow`: 页面显示时恢复播放状态，重新开始进度跟踪

### 6. 智能资源清理
根据音频播放状态智能选择清理策略：

```javascript
onUnload() {
  // 页面卸载时，如果音频正在播放，保持后台播放
  // 如果音频已停止，则清理资源
  if (this.audioContext && this.audioContext.src) {
    if (this.playing) {
      // 音频正在播放，保持后台播放，只清理页面相关资源
      this.cleanupPageResources()
    } else {
      // 音频已停止，清理所有资源
      this.cleanupAudioResources()
    }
  } else {
    // 没有音频，只清理页面资源
    this.cleanupPageResources()
  }
}
```

#### 清理策略说明
- **`cleanupPageResources()`**: 清理页面相关资源（定时器等），保持音频在后台播放
- **`cleanupAudioResources()`**: 完全清理音频资源，停止播放并移除事件监听器
- **避免错误**: 不直接清空 `BackgroundAudioManager.src`，避免 "invalid BackgroundAudioManager.src" 错误
