# 迷你播放器功能说明

## 功能概述
在首页底部显示迷你播放器，当有音频在后台播放时自动显示，提供快速控制播放的功能。

## 实现方案

### 1. 组件结构
- **位置**: `/components/MiniPlayer/index.vue`
- **集成页面**: 首页 (`/pages/home/index.vue`)

### 2. 核心功能

#### 2.1 自动检测播放状态
- 组件挂载时初始化 `BackgroundAudioManager`
- 每500ms定时检查音频状态，确保及时响应
- 监听音频播放事件（播放、暂停、停止、结束、错误）

#### 2.2 显示逻辑
- 当检测到有音频（`audioContext.src` 存在）时显示
- 音频停止或清除时自动隐藏

#### 2.3 用户交互
- **播放/暂停按钮**: 控制音频播放状态
- **关闭按钮**: 停止播放并隐藏播放器
- **点击播放器主体**: 跳转到完整播放器页面

### 3. 数据同步

#### 3.1 播放器页面设置
```javascript
// 播放器页面加载音频时保存ID
uni.setStorageSync('currentTrackId', this.trackId)

// 设置音频信息
this.audioContext.title = this.track.title || '冥想音频'
this.audioContext.coverImgUrl = this.track.coverUrl || '/static/images/default-cover.jpg'
```

#### 3.2 迷你播放器获取
```javascript
// 获取背景音频管理器
const audioContext = uni.getBackgroundAudioManager()

// 读取音频信息
this.currentTrack = {
  title: audioContext.title,
  coverUrl: audioContext.coverImgUrl,
  audioUrl: audioContext.src
}
```

### 4. 调试步骤

如果迷你播放器没有显示，请按以下步骤检查：

1. **检查控制台日志**
   - 查看是否有 "MiniPlayer: 组件创建" 和 "MiniPlayer: 组件挂载" 日志
   - 查看是否有 "MiniPlayer: 检测到音频" 或 "MiniPlayer: 没有检测到音频" 日志

2. **验证音频管理器状态**
   在播放器页面的控制台执行：
   ```javascript
   const audio = uni.getBackgroundAudioManager()
   console.log({
     src: audio.src,
     title: audio.title,
     coverImgUrl: audio.coverImgUrl,
     paused: audio.paused
   })
   ```

3. **检查组件渲染**
   - 确认首页正确引入了 MiniPlayer 组件
   - 检查组件的 `showPlayer` 计算属性是否返回 true

4. **常见问题**
   - **问题**: 播放音频后返回首页没有显示迷你播放器
   - **原因**: 可能是 BackgroundAudioManager 没有正确设置音频信息
   - **解决**: 确保播放器页面正确设置了 title、coverImgUrl 等属性

5. **测试流程**
   1. 进入播放器页面
   2. 播放音频
   3. 返回首页
   4. 应该看到底部的迷你播放器
   5. 点击播放/暂停按钮测试控制功能
   6. 点击关闭按钮测试停止功能

### 5. 样式说明

- **位置**: 固定在页面底部，距离底部 100rpx
- **背景**: 半透明白色背景，带毛玻璃效果
- **动画**: 出现时有向上滑入动画
- **阴影**: 带有阴影效果，提升视觉层次

### 6. 优化建议

1. **性能优化**
   - 定时检查间隔可根据实际需求调整（当前为500ms）
   - 可以考虑在页面隐藏时暂停定时器

2. **功能扩展**
   - 可以添加进度条显示
   - 可以添加上一曲/下一曲按钮
   - 可以添加播放列表快捷入口

3. **视觉优化**
   - 可以根据音频封面自动调整背景色
   - 可以添加播放动画效果
