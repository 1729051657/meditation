## 请求封装（按照你项目的规范，示例为 Promise 版）

```js
// utils/env.js
export const BASE_URL = 'https://your-api-host';

// utils/storage.js
export const getToken = () => wx.getStorageSync('token') || '';

// utils/request.js
import { BASE_URL } from './env';
import { getToken } from './storage';

export function request(url, method = 'GET', data = {}, extraHeaders = {}) {
  return new Promise((resolve, reject) => {
    wx.request({
      url: `${BASE_URL}${url}`,
      method,
      data,
      header: {
        'Content-Type': 'application/json',
        'Authorization': getToken() ? `Bearer ${getToken()}` : '',
        ...extraHeaders
      },
      success: ({ statusCode, data }) => {
        if (statusCode === 200 && (data.code === 200 || data.code === 0 || data.rows || data.data)) {
          resolve(data);
        } else {
          reject(data);
        }
      },
      fail: reject
    });
  });
}
```

分页参数遵循 RuoYi：`pageNum`、`pageSize`、`orderByColumn`、`isAsc`。

## 页面路由与数据流

### pages/home/index
用途：展示分类入口、推荐区块、横幅、知识列表、迷你播放器。

加载数据：
```js
import { listCategories } from '../../api/category';
import { listSlots, listSlotItems } from '../../api/recommend';
import { listBanners } from '../../api/banner';
import { listArticles } from '../../api/article';
import { listPlayHistory } from '../../api/play';

Page({
  data: { categories: [], banners: [], blocks: {}, articles: [], current: null },
  async onLoad() {
    const [cats, slots, banners, arts] = await Promise.all([
      listCategories({ parentId: 0, status: 0 }),
      listSlots({ page: 'home', status: 0 }),
      listBanners({ page: 'home', status: 0 }),
      listArticles({ status: 0, pageNum: 1, pageSize: 10 })
    ]);
    const slotItemsReq = (slots.rows || slots.data || []).map(s => listSlotItems({ slotId: s.id, status: 0 }));
    const itemsRes = await Promise.all(slotItemsReq);
    this.setData({
      categories: cats.rows || cats.data || [],
      banners: banners.rows || banners.data || [],
      blocks: (slots.rows || []).reduce((acc, s, i) => (acc[s.code] = (itemsRes[i].rows || itemsRes[i].data || []), acc), {}),
      articles: arts.rows || arts.data || []
    });
  }
});
```

## 组件与交互建议
- `banner-swiper`：接收 `banners`，支持点击跳转（外链/内容）。
- `series-card`：展示标题、封面、系列·N节；点击进详情。
- `track-card`：展示封面、标题、时长角标；点击进播放。
- `article-card`：图文卡，点击进详情。
- `mini-player`：全局挂载，显示当前曲目与播放/暂停，点击进入 `pages/player/index`。
- `chips`：分类/标签切换。

## 开发注意
- 仅展示 `status=0` 且未删除的数据。
- 时长角标：`duration_sec` → 向上取整到分钟显示“10分钟”。
- 分页：滚动触底加载下一页（携带 `pageNum+1`）。
- 鉴权：从 `storage` 读取 token 放入 `Authorization` 头；未登录场景仅限制收藏/进度/历史等需要用户态的接口。
- 进度：默认 90% 判定完成，可在前端控制阈值。

## 快速联调清单
1. 首页：分类/推荐/横幅/文章均能正常拉取并渲染；迷你播放器显示最近未完成曲目。
2. 分类页：`categoryId` 切换后，推荐位与单集列表正确刷新。
3. 系列详情：小节按 `order_index` 排序；继续播放跳转到最近小节。
4. 搜索：历史新增删除、热词显示、三类结果切换无误。
5. 收藏：单集/系列/文章三类收藏可增删，并在列表展示。
6. 播放：进度定时上报与完成状态正确，系列完成数同步更新。
