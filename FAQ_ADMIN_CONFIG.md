# FAQ数据配置说明

## 一、在后台管理系统配置（推荐）

### 1. 配置FAQ字典类型
1. 登录后台管理系统
2. 进入菜单：系统管理 -> 字典管理
3. 点击"新增"按钮，填写以下信息：
   - 字典名称：`小程序FAQ问题`
   - 字典类型：`miniapp_faq_questions`
   - 状态：正常
   - 备注：小程序常见问题配置

### 2. 配置FAQ问题数据
1. 在字典管理列表中，找到"小程序FAQ问题"
2. 点击"数据"按钮，进入字典数据管理
3. 依次添加以下问题：

| 字典标签 | 字典键值 | 字典排序 | 备注（答案） |
|---------|---------|---------|------------|
| 如何开始冥想？ | 1 | 1 | 选择一个安静的环境，采用舒适的坐姿，闭上眼睛，专注于呼吸。初学者可以从5-10分钟开始，逐渐增加时长。 |
| 冥想的最佳时间是什么时候？ | 2 | 2 | 早晨起床后和晚上睡前是最佳的冥想时间。早晨冥想可以帮助您开启美好的一天，晚上冥想有助于放松身心，改善睡眠质量。 |
| 冥想时总是走神怎么办？ | 3 | 3 | 走神是正常现象，不必自责。当您意识到走神时，温柔地将注意力带回到呼吸上。随着练习的增加，专注力会逐渐提升。 |
| 需要特殊的装备吗？ | 4 | 4 | 冥想不需要特殊装备。您只需要一个安静的空间和舒适的坐垫或椅子即可。如果愿意，可以使用冥想垫或瑜伽垫增加舒适度。 |
| 冥想有什么好处？ | 5 | 5 | 冥想可以减轻压力和焦虑，提高专注力，改善睡眠质量，增强自我意识，促进情绪稳定，提升整体幸福感。 |
| 每天需要冥想多长时间？ | 6 | 6 | 初学者可以从每天5-10分钟开始。随着练习的深入，可以逐渐增加到20-30分钟。重要的是保持规律性，而不是时长。 |
| 如何知道自己冥想是否正确？ | 7 | 7 | 没有"正确"或"错误"的冥想方式。如果您感到更加平静、专注和放松，那就是有效的冥想。每个人的体验都是独特的。 |
| 可以躺着冥想吗？ | 8 | 8 | 可以，但坐姿通常更推荐，因为躺着容易睡着。如果选择躺姿，建议保持清醒的意识，避免进入睡眠状态。 |

### 3. 配置联系方式
1. 进入菜单：系统管理 -> 参数设置
2. 点击"新增"按钮，依次添加以下参数：

| 参数名称 | 参数键名 | 参数键值 | 备注 |
|---------|---------|---------|------|
| 小程序客服邮箱 | miniapp.contact.email | support@meditation.com | 小程序FAQ页面显示的客服邮箱地址 |
| 小程序客服电话 | miniapp.contact.phone | 400-123-4567 | 小程序FAQ页面显示的客服电话 |
| 小程序客服微信 | miniapp.contact.wechat | meditation_service | 小程序FAQ页面显示的客服微信号 |
| 小程序客服QQ | miniapp.contact.qq | 88888888 | 小程序FAQ页面显示的客服QQ号 |
| 小程序联系地址 | miniapp.contact.address | 北京市朝阳区xxx大厦8层 | 小程序FAQ页面显示的联系地址 |

## 二、使用SQL脚本配置

提供了三个SQL脚本文件：

1. **FAQ_INIT_DATA.sql** - 完整版初始化脚本，包含详细注释
2. **FAQ_QUICK_INIT.sql** - 简化版快速初始化脚本
3. **FAQ_SNOWFLAKE_ID.sql** - 使用雪花ID的版本

### 执行步骤：
```bash
# 1. 连接到数据库
mysql -u root -p your_database

# 2. 执行SQL脚本（选择其中一个）
source /path/to/FAQ_QUICK_INIT.sql
```

## 三、数据说明

### FAQ数据结构
- **字典标签（dict_label）**：显示的问题文本
- **字典键值（dict_value）**：问题的唯一ID
- **字典排序（dict_sort）**：控制显示顺序
- **备注（remark）**：问题的答案内容

### 联系方式数据结构
- **参数键名（config_key）**：系统识别的唯一键
- **参数键值（config_value）**：实际显示的值
- **参数名称（config_name）**：后台显示的名称
- **备注（remark）**：参数说明

## 四、注意事项

1. **ID冲突处理**
   - 如果使用SQL脚本，确保ID不与现有数据冲突
   - 建议使用后台管理系统添加，系统会自动生成ID

2. **租户处理**
   - 默认租户ID为 `000000`
   - 如需其他租户，请修改tenant_id字段

3. **权限要求**
   - 需要有字典管理和参数设置的权限
   - 默认admin用户拥有所有权限

4. **数据更新**
   - 可以随时在后台修改问题和答案
   - 修改后小程序会实时获取最新数据
